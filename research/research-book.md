# Book — Summary & Dossier

Research a specific book given its title, author, or other identifying details. Produce a book summary file in the BOOK Summary folder.

## Workflow

1. **Identify the book** — confirm the exact title, author, and edition from the user's description. If ambiguous, search and ask the user to confirm before proceeding.
2. **Search broadly** — use web search to find summaries, reviews, chapter lists, publication details, and author background
3. **Cross-reference** — verify facts across multiple sources (publisher page, Wikipedia, Goodreads, review sites)
4. **Write the book file** — using the format below, saved to the BOOK Summary folder
5. **No RSH Log entry** — book summaries live in BOOK Summary, not RSH Log

## File Location

```
/Users/oblinger/ob/kmr/Topic/Misc/BOOK/BOOK Summary/{YYYY-MM-DD} {Title} by {Author}.md
```

- Date is the date the summary was created
- Title uses title case
- Author is the primary author's full name

## Book Summary Format

```markdown
# {Book Title}

**Author:** {Author name(s)}
**Published:** {Year}
**Pages:** {Page count}
**Genre:** {Genre(s)}
**Awards:** {Notable awards, if any — omit section if none}

## Summary

{2-3 paragraphs: what the book is about, its central thesis, and why it matters. Written for someone who hasn't read it.}

## Core Arguments

{3-5 bold-header paragraphs, each covering a major argument or theme of the book. Format: **Claim.** Explanation.}

## Chapter Overview

{Numbered list of chapters with title and a brief (one-line) description of each. If the book has many chapters, group by part/section.}

## Key Takeaways

{Bulleted list of 4-6 takeaways — the most important ideas a reader should walk away with.}

## About the Author

{1 paragraph: who they are, credentials, other notable works.}

## Sources

{Full URLs to all referenced material — publisher page, author site, review sites, etc.}
```

## Tips

- Prefer primary sources (publisher page, author's own site) over third-party summaries
- Include chapter titles even if descriptions are brief — they help readers decide whether to read specific sections
- For academic/technical books, note the intended audience and prerequisite knowledge
- For older or classic books, note historical context and lasting influence
- Keep the summary accessible to a general reader even for technical books
